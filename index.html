Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It describes the structure of web pages using markup.

Note: HTML is not a programming language; it's a markup language that defines the structure of your content.

What You'll Learn

  • The basic structure of HTML documents
  • HTML elements and attributes
  • How to create headings, paragraphs, and text formatting
  • Working with links, images, lists, and tables
  • Creating forms for user input
  • Semantic HTML for better structure and accessibility
  • Advanced features like multimedia and graphics

History of HTML

HTML was created by Tim Berners-Lee in 1991. The first version had only 18 elements. Since then, HTML has evolved through several versions:

Version Year Significant Changes
HTML 2.0 1995 First standardized version
HTML 3.2 1997 Added tables, applets, text flow around images
HTML 4.01 1999 Added stylesheets, scripting, frames
XHTML 1.0 2000 Stricter XML-based version of HTML
HTML5 2014 Added semantic elements, multimedia, graphics APIs
Simple HTML Document
<!DOCTYPE html>
<html>
<head>
    <title>Page Title</title>
</head>
<body>
    <h1>My First Heading</h1>
    <p>My first paragraph.</p>
</body>
</html>

HTML Basics

HTML documents are made up of elements that describe the structure of the page. These elements are represented by tags that mark up content.

Key Concepts

  • Elements: The building blocks of HTML pages
  • Tags: Markers that define elements
  • Attributes: Provide additional information about elements
  • Nesting: Placing elements inside other elements
Tip: Most HTML elements have an opening tag, content, and a closing tag. Some elements are self-closing.

HTML Document Structure

Every HTML document has a basic structure that must be followed. This includes document type declaration, HTML element, head section, and body section.

Basic Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>

Document Type Declaration

The <!DOCTYPE html> declaration defines the document type and HTML version (HTML5 in this case).

HTML Element

The <html> element is the root element and contains all other HTML elements.

Head Section

The <head> element contains meta information about the document, including:

  • <title> - Sets the title shown in browser tab
  • <meta> - Provides metadata like character set
  • <link> - Links to external resources (CSS)
  • <style> - Contains CSS styles
  • <script> - Contains JavaScript code

Body Section

The <body> element contains the visible page content.

HTML Elements

HTML elements are the building blocks of HTML pages. Each element has a specific purpose and meaning.

Element Syntax

Most HTML elements have an opening tag, content, and a closing tag:

<tagname>content goes here...</tagname>

Some elements are empty (self-closing) and don't have content or a closing tag:

<tagname>

or in XHTML style:

<tagname />

Common HTML Elements

Element Description Example
<h1> to <h6> Heading elements <h1>Main Heading</h1>
<p> Paragraph <p>This is a paragraph.</p>
<a> Anchor (link) <a href="page.html">Link</a>
<img> Image <img src="image.jpg" alt="Description">
<div> Division/container <div>Content</div>
<span> Inline container <span>text</span>
Note: HTML5 introduced many new semantic elements like <header>, <footer>, <article>, and <section> which we'll cover later.

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the start tag.

Attribute Syntax

<tagname attribute="value">content</tagname>

Common Attributes

Attribute Description Example
id Unique identifier for an element <div id="header">
class Specifies one or more class names for an element <p class="intro">
style Inline CSS styles <p style="color:red;">
src Specifies the source URL for media elements <img src="image.jpg">
href Specifies the URL for links <a href="page.html">
alt Alternative text for images <img alt="Description">
title Extra information about an element (tooltip) <a title="Tooltip">

Boolean Attributes

Some attributes don't need a value - their presence implies true:

<input type="checkbox" checked>
<button disabled>Click</button>
Warning: Always quote attribute values. While HTML5 allows unquoted values in some cases, it's better practice to always use quotes.

Headings & Paragraphs

Headings and paragraphs are the most basic text elements in HTML.

Headings

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important).

Heading Example
<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Sub-subheading</h3>
<h4>Level 4 Heading</h4>
<h5>Level 5 Heading</h5>
<h6>Level 6 Heading</h6>

Best Practice: Use headings semantically to structure your content, not just for visual styling. There should only be one <h1> per page.

Paragraphs

The <p> element defines a paragraph:

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

Line Breaks

The <br> element creates a line break:

<p>This is the first line.<br>This is the second line.</p>

Horizontal Rules

The <hr> element creates a thematic break (horizontal line):

<p>Section one content.</p>
<hr>
<p>Section two content.</p>

Quotations

HTML provides elements for marking up quotations:

  • <blockquote> - For long quotations (block-level)
  • <q> - For short quotations (inline)
  • <cite> - For citations
Quotation Example
<blockquote cite="https://example.com">
    This is a long quotation from another source.
</blockquote>

<p>As <cite>Albert Einstein</cite> said: <q>Imagination is more important than knowledge.</q></p>

Computer Code

Elements for displaying computer code:

  • <code> - Inline code snippet
  • <pre> - Preformatted text (preserves whitespace)
  • <kbd> - Keyboard input
  • <samp> - Sample output
  • <var> - Variable

Links & Anchors

The <a> (anchor) element creates hyperlinks to other web pages, files, locations within the same page, or any other URL.

Basic Link Syntax

<a href="url">link text</a>

Link Types

Links to other websites:

<a href="https://www.example.com">Visit Example</a>

Links to other pages on your site (relative URLs):

<a href="about.html">About Us</a>
<a href="/products/index.html">Products</a>

Links to sections within the same page:

<!-- First create an anchor point with id -->
<h2 id="section1">Section 1</h2>

<!-- Then link to it -->
<a href="#section1">Jump to Section 1</a>

Email links that open the user's email client:

<a href="mailto:someone@example.com">Send Email</a>
<a href="mailto:someone@example.com?subject=Hello&body=Message">Pre-filled Email</a>

Links that trigger a download:

<a href="document.pdf" download>Download PDF</a>
<a href="document.pdf" download="new-filename.pdf">Download with New Name</a>

Link Attributes

Attribute Description
href Specifies the URL the link goes to
target Specifies where to open the linked document (_blank, _self, _parent, _top)
rel Specifies the relationship between documents (nofollow, noopener, etc.)
download Specifies that the target will be downloaded
Link Example with Attributes
<a href="https://www.example.com" target="_blank" rel="noopener noreferrer">
    Visit Example in New Tab
</a>
Security: When using target="_blank", always include rel="noopener noreferrer" to prevent potential security vulnerabilities.

Images

The <img> element is used to embed images in HTML pages. It's an empty element (no closing tag).

Basic Syntax

<img src="image.jpg" alt="Description">

Key Attributes

Attribute Description
src Path to the image file (required)
alt Alternative text for accessibility (required)
width Width of the image in pixels
height Height of the image in pixels
title Tooltip text shown on hover
loading Lazy loading ("lazy" or "eager")
Image Example
<img src="nature.jpg" alt="Beautiful nature landscape" 
     width="600" height="400" title="Nature Photo">

Responsive Images

HTML5 introduced the <picture> element for responsive images:

<picture>
    <source media="(min-width: 1200px)" srcset="large.jpg">
    <source media="(min-width: 768px)" srcset="medium.jpg">
    <img src="small.jpg" alt="Responsive image">
</picture>

Image Maps

Create clickable areas on an image:

<img src="workplace.jpg" alt="Workplace" usemap="#workmap">

<map name="workmap">
    <area shape="rect" coords="34,44,270,350" href="computer.html" alt="Computer">
    <area shape="circle" coords="337,300,44" href="coffee.html" alt="Coffee">
</map>
Best Practice: Always provide meaningful alt text for accessibility and SEO. For decorative images, use alt="".

Lists

HTML provides several types of lists for organizing content.

Unordered Lists

Created with <ul> (unordered list) and <li> (list item) elements:

<ul>
    <li>Coffee</li>
    <li>Tea</li>
    <li>Milk</li>
</ul>

Ordered Lists

Created with <ol> (ordered list) and <li> elements:

<ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

Ordered List Attributes

Attribute Description
type Numbering type (1, A, a, I, i)
start Starting number
reversed Reverses the numbering
Ordered List Example
<ol type="A" start="3">
    <li>Item C</li>
    <li>Item D</li>
    <li>Item E</li>
</ol>

Description Lists

Created with <dl> (description list), <dt> (term), and <dd> (description):

<dl>
    <dt>Coffee</dt>
    <dd>- black hot drink</dd>
    <dt>Milk</dt>
    <dd>- white cold drink</dd>
</dl>

Nested Lists

Lists can be nested inside other lists:

<ul>
    <li>Coffee</li>
    <li>Tea
        <ul>
            <li>Black tea</li>
            <li>Green tea</li>
        </ul>
    </li>
    <li>Milk</li>
</ul>
Tip: Use CSS to control list styling (bullet types, numbering, indentation, etc.).

<caption> Defines a table caption
<thead> Groups header content

Forms & Input

HTML forms are used to collect user input. The <form> element defines a form.

Basic Form Structure

<form action="/submit" method="post">
    <!-- Form elements go here -->
    <input type="submit" value="Submit">
</form>

Form Attributes

Attribute Description
action URL where the form data is sent
method HTTP method (get or post)
enctype Encoding type for file uploads (multipart/form-data)
target Where to display response (_blank, _self, etc.)
autocomplete Enable/disable autocomplete (on/off)
novalidate Disable form validation

Input Elements

The <input> element is the most important form element. Its type is defined by the type attribute.

Form Validation

HTML5 introduced built-in form validation:

Attribute Description

min, max Minimum/maximum value for numbers
pattern Regular expression pattern
Validation Example
<input type="text" required minlength="3" maxlength="20">
<input type="number" min="18" max="99">
<input type="email" required>
<input type="password" pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}" 
       title="Must contain at least one number, one uppercase and lowercase letter, and at least 8 characters">

Semantic HTML

Semantic HTML introduces meaning to the web page rather than just presentation. It makes the content more accessible and SEO-friendly.

Why Semantic HTML?

  • Better accessibility for screen readers
  • Improved SEO (search engines understand content better)
  • Easier to maintain and understand code
  • Future-proofing your website

Semantic Elements

Element Description
<header> Introductory content or navigation
<nav> Navigation links
<main> Main content of the document
<article> Self-contained composition
<section> Thematic grouping of content
<aside> Content aside from main content
<footer> Footer for a section or page
<figure> Self-contained content like images
<figcaption> Caption for a figure
<time> Machine-readable date/time
<mark> Highlighted text

Semantic Layout Example

<!DOCTYPE html>
<html>
<head>
    <title>Semantic Example</title>
</head>
<body>
    <header>
        <h1>Website Title</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Article Title</h2>
            <p>Article content...</p>
            <section>
                <h3>Comments</h3>
                <p>First comment...</p>
            </section>
        </article>
        
        <aside>
            <h2>Related</h2>
            <ul>
                <li><a href="#">Related article 1</a></li>
            </ul>
        </aside>
    </main>
    
    <footer>
        <p>© 2023 Company Name</p>
    </footer>
</body>
</html>
Tip: Use semantic elements appropriately based on the meaning of the content, not how they look. CSS controls appearance.

Graphics

HTML5 provides several ways to draw graphics directly in the browser.

Canvas

The <canvas> element is used to draw graphics via JavaScript:

<canvas id="myCanvas" width="200" height="100">
    Your browser does not support canvas.
</canvas>

<script>
    const canvas = document.getElementById('myCanvas');
    const ctx = canvas.getContext('2d');
    ctx.fillStyle = 'red';
    ctx.fillRect(10, 10, 150, 80);
</script>

Canvas Drawing Methods

Method Description
fillRect() Draws a filled rectangle
strokeRect() Draws a rectangular outline
clearRect() Clears the specified area
fillText() Draws filled text
strokeText() Draws text outline
beginPath() Begins a new path
moveTo() Moves the path to a point
lineTo() Adds a line to the path
arc() Adds an arc to the path
fill() Fills the current path
stroke() Draws the current path

SVG

SVG (Scalable Vector Graphics) defines vector-based graphics in XML format:

<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>

Common SVG Elements

Element Description
<circle> Draws a circle
<rect> Draws a rectangle
<line> Draws a line
<polygon> Draws a polygon
<path> Draws complex paths
<text> Draws text
Comparison: Canvas is pixel-based (good for games, image manipulation) while SVG is vector-based (good for logos, icons, scalable graphics).

HTML APIs

HTML5 introduced several JavaScript APIs that enhance web applications.

Geolocation API

Gets the geographical position of a user:

if (navigator.geolocation) {
    navigator.geolocation.getCurrentPosition(
        function(position) {
            console.log("Latitude: " + position.coords.latitude);
            console.log("Longitude: " + position.coords.longitude);
        },
        function(error) {
            console.error("Error getting location: " + error.message);
        }
    );
} else {
    console.log("Geolocation is not supported by this browser.");
}

Drag and Drop API

Makes elements draggable and droppable:

<div id="drag" draggable="true" ondragstart="drag(event)">Drag me</div>
<div id="drop" ondrop="drop(event)" ondragover="allowDrop(event)">Drop here</div>

<script>
    function allowDrop(e) {
        e.preventDefault();
    }
    
    function drag(e) {
        e.dataTransfer.setData("text", e.target.id);
    }
    
    function drop(e) {
        e.preventDefault();
        const data = e.dataTransfer.getData("text");
        e.target.appendChild(document.getElementById(data));
    }
</script>

Web Storage API

Stores data locally in the browser (covered in next section).

Web Workers

Runs JavaScript in the background without blocking the UI:

// main.js
const worker = new Worker("worker.js");
worker.postMessage("Start working");
worker.onmessage = function(e) {
    console.log("Worker says: " + e.data);
};

// worker.js
onmessage = function(e) {
    console.log("Message from main: " + e.data);
    // Do intensive task
    postMessage("Work done");
};

Server-Sent Events

Receives automatic updates from a server:

const source = new EventSource("updates.php");
source.onmessage = function(e) {
    console.log("Update: " + e.data);
};

WebSocket API

Creates a persistent connection between client and server:

const socket = new WebSocket("ws://example.com/socket");
socket.onopen = function(e) {
    socket.send("Hello Server!");
};
socket.onmessage = function(e) {
    console.log("Message from server: " + e.data);
};

Web Storage

The Web Storage API provides mechanisms for storing data in the browser.

Local Storage vs Session Storage

Feature Local Storage Session Storage
Scope Persists across browser sessions Limited to the current session
Capacity ~5MB ~5MB
Access Accessible from any window/tab of the same origin Accessible only within the same tab

Web Storage Methods

Method Description
setItem(key, value) Stores a key/value pair
getItem(key) Retrieves a value by key
removeItem(key) Removes a key/value pair
clear() Clears all stored data
key(index) Gets the key at a specific index

Usage Examples

// Store data
localStorage.setItem('username', 'john_doe');
localStorage.setItem('theme', 'dark');

// Retrieve data
const username = localStorage.getItem('username');
const theme = localStorage.getItem('theme');

// Remove item
localStorage.removeItem('theme');

// Clear all
localStorage.clear();

// Store object (need to stringify)
const user = { name: 'John', age: 30 };
localStorage.setItem('user', JSON.stringify(user));

// Retrieve object
const storedUser = JSON.parse(localStorage.getItem('user'));
// Store data
sessionStorage.setItem('session_id', 'abc123');

// Retrieve data
const sessionId = sessionStorage.getItem('session_id');

// Remove item
sessionStorage.removeItem('session_id');

// Clear all
sessionStorage.clear();
Security: Web Storage is not secure for sensitive data as it's accessible via JavaScript. Use HTTP-only cookies for sensitive information.

HTML Tables

HTML tables allow you to arrange data into rows and columns.

Note: While tables are great for tabular data, they should not be used for page layout. Use CSS for layout instead.

Basic Table Structure

<table>
    <tr>
        <th>Header 1</th>
        <th>Header 2</th>
    </tr>
    <tr>
        <td>Row 1, Cell 1</td>
        <td>Row 1, Cell 2</td>
    </tr>
    <tr>
        <td>Row 2, Cell 1</td>
        <td>Row 2, Cell 2</td>
    </tr>
</table>
Basic Table Example
Header 1 Header 2
Row 1, Cell 1 Row 1, Cell 2
Row 2, Cell 1 Row 2, Cell 2

Table Elements

Element Description
<table> Defines a table
<tr> Defines a table row
<th> Defines a table header cell
<td> Defines a table data cell
<caption> Defines a table caption
<thead> Groups header content
<tbody> Groups body content
<tfoot> Groups footer content
<colgroup> Specifies column groups
<col> Specifies column properties

Advanced Table Example

<table>
    <caption>Monthly Savings</caption>
    <colgroup>
        <col span="2" style="background-color:#f1f1f1">
    </colgroup>
    <thead>
        <tr>
            <th>Month</th>
            <th>Savings</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>January</td>
            <td>$100</td>
        </tr>
        <tr>
            <td>February</td>
            <td>$80</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td>Total</td>
            <td>$180</td>
        </tr>
    </tfoot>
</table>

Table Attributes

Attribute Description
colspan Number of columns a cell should span
rowspan Number of rows a cell should span
scope Specifies if header cell applies to row, column, or group
headers Associates data cells with header cells
